HOW IT WORKS:

Proof Caching (Phase A):
  1. User generates proof from reasoning
  2. System hashes reasoning deterministically
  3. Check cache: hash found?
     - YES → return cached proof instantly (0ms)
     - NO → generate proof (62ms) + cache it
  4. Track usage for analytics

Data Deduplication (Phase B):
  1. User uploads data (CSV, JSON, etc)
  2. System hashes content (content-hash)
  3. Check ecosystem: hash exists?
     - YES → deduplication detected!
       - User gets reference to existing asset
       - No new storage, no new computation
       - Both users can access same asset
     - NO → register as new asset
       - Store asset metadata + ownership
       - Start tracking usage
  4. When user accesses asset:
     - Increment usage counter
     - Record in usage table
     - Ready for Phase C revenue

Revenue Recycling (Phase C - ready to implement):
  1. User A owns asset (uploaded data)
  2. User B uses asset
  3. System detects usage
  4. Credit transfer:
     - User B pays: 0.75 credits
     - User A gets: 0.50 credits
     - Platform gets: 0.25 credits
  5. User A sees earnings accumulate
